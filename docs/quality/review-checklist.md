# Review Checklist

## 一、架构检查

- [ ] Controller 是否只负责接收请求和返回响应
- [ ] Controller 是否没有直接调用 Mapper
- [ ] Service 是否包含业务逻辑
- [ ] Mapper 是否只负责数据库访问
- [ ] Entity 是否没有直接暴露给前端
- [ ] DTO、VO、Entity 是否职责清晰

## 二、数据库检查

- [ ] 表字段是否符合 docs/design/database.md
- [ ] 是否使用逻辑删除
- [ ] 查询是否过滤 deleted = 0
- [ ] employeeNo 是否有唯一约束
- [ ] 状态字段是否符合枚举定义
- [ ] 是否存在未说明的字段

## 三、接口检查

- [ ] 接口路径是否符合 docs/specs/api.md
- [ ] 请求参数是否有校验
- [ ] 返回格式是否统一
- [ ] 错误码是否明确
- [ ] 是否没有把异常堆栈直接返回给前端

## 四、代码质量检查

- [ ] 命名是否清晰
- [ ] 是否有重复代码
- [ ] 方法是否过长
- [ ] 是否有无用代码
- [ ] 是否有硬编码配置
- [ ] 是否有必要注释

## 五、安全检查

- [ ] 数据库密码是否没有写死
- [ ] 日志是否没有打印密码、token、cookie
- [ ] 参数是否经过校验
- [ ] 是否避免 SQL 注入风险

## 六、测试检查

- [ ] 是否有核心 Service 测试
- [ ] 是否测试 employeeNo 重复
- [ ] 是否测试逻辑删除
- [ ] 是否测试分页查询
- [ ] 是否提供手动验证步骤

## 七、Sprint 检查

- [ ] 是否满足当前 Sprint Contract
- [ ] 是否没有扩大需求范围
- [ ] 是否更新 docs/progress.md
